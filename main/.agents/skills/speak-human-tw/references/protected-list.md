# 保護清單與誤殺防護

去 AI 味最大的風險不是改不乾淨，是把事實一起改掉。動筆前圈出這些內容、全程原封不動，改完逐項核對。

## 五類保護對象

**1. 價格與數字**：定價、優惠碼、折扣、名額、日期、統計數據、成效指標。逐字保留；精確度不漂移（「約 42%」不能變「超過四成」）；**絕不為了「更具體」補原文沒有的數字**。

**2. 專有名詞**：課程名、品牌名、產品名、活動名。不因「換詞循環」被同義替換（《超級個體工作術》第二次出現還是原名，不會變「這門課程」）；中國平台專名照原文，不套用替換表。

**3. 網址與連結文字**：連結本體與錨點文字保留。例外——AI 工具參數（`utm_source=chatgpt.com`、`referrer=grok.com`）要清；作者自設的行銷 UTM（`utm_source=newsletter`）保留。

**4. 真實姓名與引號內原話**：真名不可改成「這位學員」等模糊代稱；引號內原話一字不改（含裡面的中國用語、半形標點、AI 味句型——引用的價值就是原汁原味）；原話有錯字標註提醒，不代改。

**5. 承諾類文字**：退費、保固、免責、服務範圍。只能調前後銜接語氣，不能調意思；「14 天內」「全額」這類量詞條件一字不漂；金流制式語（「依發卡銀行作業為準」）是準確性需求不是罐頭腔。

## 容易誤殺的情況（先放行）

| 看起來像 | 其實是 | 判斷方法 |
| :-- | :-- | :-- |
| 排比三段式 | 承載真實對比的修辭 | 底下有沒有具體事實接著 |
| 無源引用 | 有來源的數據 | 「根據我們後台的數據」有出處 |
| 「不是 A 而是 B」 | 全文唯一的核心轉折 | 規則是整篇最多一次，非零容忍 |
| 客服罐頭腔 | 金流／條款必要制式語 | 涉及錢、法律、責任的制式語保留 |
| 重複句式 | 長文的節奏設計 | 重複有沒有在推進敘事 |
| AI 味詞彙 | 被討論的對象 | 「我戒掉『賦能』」是提及不是使用 |
| 正式官腔 | 公告場景合宜語域 | 停機公告本來就該正式 |
| 空泛的「優化」 | 掛著具體動作與指標 | 後面接得出「改了什麼、變化多少」 |
| 口語碎片、不完整句 | 人味本體 | 這是要保護的方向，一字不動 |

拿不準的通用判斷法：**這句刪掉或改掉後，讀者少知道了什麼？** 少了任何事實、立場或情緒就不能刪；什麼都沒少才是空話。

## 回讀核對程序

1. 把圈出的保護對象列成清單（價格 N 項、名詞 N 項、連結 N 項、真名原話 N 項、承諾 N 項）。
2. 在改寫版逐項搜尋確認存在且逐字一致。
3. 確認沒有新增原文沒有的事實、數字、來源。
4. 確認作者立場沒翻轉或稀釋（「我反對」不能變「我持保留態度」）。
5. 任何一項失敗：回去修正，不交稿。
